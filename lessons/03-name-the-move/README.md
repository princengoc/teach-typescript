# Lesson 03: name the move

**Start here:** look at the preview panel on the right.

1. **Try the move** first. Paint one shelf with the buttons and count your
   presses. Then press "Teach the robot this move" and paint the next shelf
   with one press. Same calls, one name.
2. **Paint the shelves**, by finishing `paintShelf` in `src/exercise.ts` (open
   on the left). The three calls at the bottom of the file are written for
   you; your work is inside the function. Save and watch the preview replay it.

When the verdict turns green and reads `PASS`, the shelves are done. You do
not need the terminal for this lesson. The preview is your test.

## Then the drills

`drills/` holds five small jobs with no robot in them. Start at `01` and work
down. Each has a `readme.md` saying what to do and a `hints.md` if you get
stuck. Run one with `npx vitest run drills/01-what-it-says` in the terminal,
or just watch the errors in the editor.

Come back to the preview for the card when you are through.

## Rules

- Edit `src/exercise.ts` and the files in `drills/` only. `src/harness/` runs
  the world; leave it closed.
- A bug inside `paintShelf` shows up on all three shelves at once. That is not
  three bugs. Fix it once.
- `wordbook.md` lists every word so far. Look there before asking.
