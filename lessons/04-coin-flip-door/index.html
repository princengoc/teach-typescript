Lesson 4: the coin-flip door

You know what an if and an else do. This lesson is how TypeScript spells them. 

if (hasKey) {
  openTheDoor();
} else {
  knock();
}
  • The question goes in round brackets, right after if.
  • Each block goes in curly braces. No then, no colon.
  • else sits on the same line as the } that closed the first block.
  • The question has to be true or false. A value that is only ever one of those two is called a boolean.
  • To ask whether two values are the same, write ===. One = means something else, and TypeScript will tell you so.

Now here is something to spend it on.

The room

One shelf to paint. Half the time the door is in the left wall, half the time in the right, and you do not get to choose. Either way the robot walks in standing on the door, facing down the page.

Here is a move that turns right, then paints three squares in a row. Try it in both rooms. 

robot.turnRight();
robot.paint();
robot.walk(1);
robot.paint();
robot.walk(1);
robot.paint();

One room painted, one crash. The two rooms need two different turns, and this code cannot tell the rooms apart.

So the robot gets a question. robot.wallOnLeft(); hands back a boolean: true when the square on the robot's left is a wall, false when it is open floor. Ask it in each room.

Two rooms, two answers. That question is a boolean, so it goes straight in the round brackets of an if.
